theory

For a long time, we've built sites that specified column widths in HTML. This leads to a compromised design, with an app or site that looks good on a single large resolution, and reasonable on mobile. It also produces HTML stuffed with non-semantic class names.

This isn't good enough.

The Open Digital Application Wireframing Styleset (or OpenDAWS) is an attempt to correct this.

what OpenDAWS is...

OpenDAWS can be thought of as an structured library; that is to say, it's a huge set of resets and helpful classes, as well as the logic for grid layouts, but without any of the constraints of a full framework. There's no fixed grid you have to adhere to. Instead, you simply apply widths to containing element IDs.

This means that you can create a design limited in width only by the user's viewport size. Want a design that adapts at 320px, 768px, 1024px, 1600px, 1920px, 2560px & 4096px? Easy!

Moreover, by not specifying widths in HTML, we're free to completely change how our design looks at these sizes. Thus an element that's 4 columns width at 320px can be 8 columns width at 1024px, or 9 columns at 1920px, or 16 columns at 4096px. The limit is your design knowledge, rather than fixed HTML.

...& what it isn't

This is not a full framework, in the vein of Foundation, Bootstrap or Skeleton. Those all come with a set grid, which this doesn't. As such, if you're new to design, I'd suggest going with one of those; they'll give you a firmer foundation on which to learn.

It's also not an attempt to style everything. Whilst OpenDAWS includes a full set of typographic stacks, and some pleasant defaults for buttons and forms, it's designed to be extended. As such, you're going to need to write a lot of CSS on top of it.

Finally, it's not backwards incompatible. Everything's supported back to IE6, through Modernizr and selectivizr powered with Mootools. Feel free to use [attr=], :first/last/only-child, :focus and so on. For added support, you can team it up with NWMatcher or MooTools, for complete IE6+ pseudo class support!

benefits

  • Normalised elements for x-broser consistency
  • Baseline aligned typography and forms
  • Automatically responsive type sizes for different resolution sizes
  • Semantically-correct HTML
  • Designs can be designed for each new resolution, rather than set inflexibly in HTML
  • Mobile-first design enforced

features

  • A full set of sans, serif and alternative font stacks, ordered by likely font appearance
  • All the benefits of LESS, plus the LESShat mixin library
  • Beautiful defaults form forms and typography
  • Automatically x-browser compliant grid functionality
  • Box sizing by default to solve padding increasing size bugs
  • Fontawesome built in as the default icon library

Example Code

basic usage

html

<header>...</header>
<section>
<article>...</article>
<aside>
<div id="sidebar1">...</div>
<div id="sidebar2">...</div>
</aside>
</section>

mobile css

section { .row(); .width(6) }
header, article, aside { .col(); .width(6) }

making it responsive

desktop css

@media screen and (min-width: 50em) {
header { .width(12); }
article { .width(9); }
aside { .width(3); .gutter(); }
}

hd css

@media screen and (min-width: 86em) {
header { .width(20); }
article { .width(10); }
aside { .width(10); }
sidebar1 { .width(5); }
sidebar2 { .width(5); .gutter(); }
}
